> [!用語] 宣言型プロセスの自動化（Declarative Process Automation）
>
> コードを書かず**画面設定だけ**で業務処理を自動化する機能群。代表例は **フロー（Flow）／承認プロセス（Approval Process）**。条件付きの項目更新や承認依頼をクリック操作で作れる。

> [!用語] Apex（エイペックス）
>
> Salesforce の**サーバー側プログラミング言語**。Java に似た構文で、データベース操作（SOQL/DML）を言語に組み込んでいる。宣言型で実現できない複雑なロジックに使う。

> [!用語] 変数・定数・メソッド（Apex の基本パーツ）
>
> - **変数（Variable）**：値を入れる箱。型を指定して宣言（例：`Integer count = 0;`）。
> - **定数（Constant）**：変えない値。`final` を付けて宣言。
> - **メソッド（Method）**：処理をまとめた部品。引数を受け取り戻り値を返せる。

```apex
public class OrderCalculator {
    // 定数：消費税率（finalを付けると後から変更できない）
    public static final Decimal TAX_RATE = 0.10;

    // メソッド：税込金額を計算して返す
    public static Decimal calcTaxIncluded(Decimal price) {
        Decimal taxIncluded = price * (1 + TAX_RATE);
        return taxIncluded;
    }
}
```

> [!用語] フロー制御ステートメント（Flow Control Statement）
>
> 処理の流れを制御する命令。分岐の `if / else`、繰り返しの `for / while`、多分岐の `switch` など。宣言型機能の「フロー」とは別物なので混同しない。

```apex
// フロー制御の例：if 分岐 と for ループ
List<Account> accounts = [SELECT Id, Name, AnnualRevenue FROM Account];
for (Account acc : accounts) {              // for：全件を繰り返し
    if (acc.AnnualRevenue > 1000000) {      // if：条件で分岐
        acc.Rating = 'Hot';
    } else {
        acc.Rating = 'Warm';
    }
}
```

> [!例] 宣言型とプログラミング型のどちらを選ぶ?
>
> - 「商談成立で取引先の項目を更新」程度なら **フロー（宣言型）** で十分。
> - 「複数オブジェクトをまたぐ複雑な計算」「大量レコードの一括処理」「外部連携」なら **Apex（プログラミング型）**。
> - 試験では「**まず宣言型で実現できないか検討する**」が前提。

## 試験対策：押さえておきたいポイント

> [!ポイント] Apex の基本構成概念チェックリスト
>
> | 概念 | 要点 |
> | --- | --- |
> | 変数 | 型を指定して宣言（`Integer`、`String`、`Decimal`、`List` など） |
> | 定数 | `final` で宣言、後から変更不可 |
> | メソッド | 引数を受け取り戻り値を返す。`void` は戻り値なし |
> | if / else | 条件分岐 |
> | for / while | 繰り返し（ループ） |
> | switch | 値による多分岐 |

> [!注意] 「フロー」と「フロー制御」は別物
>
> **フロー（Flow）= 宣言型の自動化機能**、**フロー制御ステートメント = Apex コード内の if/for などの制御命令**。文脈で見分ける。
