# 取引先責任者リストを表示およびプレビューする

## 学習の目的

ここまでで取引先責任者データの取得まで完了しました。最後のステップでは、取得したデータを **`lightning:datatable` で表示** し、表示する **列（カラム）の定義** を JavaScript 側で設定して、クイックスタート全体を完成させます。

> [!ポイント] このユニットで覚えること
>
> - `lightning:datatable` は **リストを表形式で表示** する基本コンポーネントであること。
> - `data`（表示データ）・`columns`（列定義）・`keyField`（行を一意に識別する項目）の3点が必須であること。
> - 列定義は **`{label, fieldName, type}`** のオブジェクトの配列で指定すること。

---

## 取引先責任者リストを表示する

前ユニットで入れたコメント `<!-- Contact list goes here -->` の場所にデータテーブルを差し込みます。

> [!手順] データテーブルをマークアップに追加する
>
> 1. コメント `<!-- Contact list goes here -->` の後の新しい行に次のコードを追加する。
> 2. **[File（ファイル）] | [Save（保存）]** を選択する。

```html
<lightning:datatable data="{! v.Contacts }" columns="{! v.Columns }" keyField="Id" hideCheckboxColumn="true"/>
```

> [!用語] lightning:datatable（データテーブル）
>
> 行と列からなる表を表示する Lightning 基本コンポーネント。SLDS 準拠の見やすい表を少ない記述で作れ、並べ替えやインライン編集などの機能も備えます。

`lightning:datatable` の各属性の意味は次のとおりです。

| 属性 | 設定値 | 役割 |
| --- | --- | --- |
| `data` | `{! v.Contacts }` | 表に表示する **データ**（取得した取引先責任者リスト） |
| `columns` | `{! v.Columns }` | 表示する **列の定義**（このあと JavaScript で設定） |
| `keyField` | `"Id"` | 各行を **一意に識別** する項目。Contact の `Id` を使用 |
| `hideCheckboxColumn` | `"true"` | 行頭の **チェックボックス列を非表示** にする |

> [!注意] keyField は必須
>
> `lightning:datatable` は各行の区別に `keyField` を必要とします。一意に識別できる項目（通常は `Id`）を指定しないと、正しく描画されない・警告が出る場合があります。

## 列を定義する

データテーブルに「何の列を、どんな見出しで、どんな型で表示するか」を JavaScript コントローラー側で設定します。

> [!手順] JavaScript コントローラーに列定義を追加する
>
> 1. **[MyContactListController.js]** タブをクリックする。
> 2. `var action` 行の **上** の新しい行に次のコードを追加して `myAction` 関数を更新する。
> 3. **[File（ファイル）] | [Save（保存）]** を選択する。
> 4. 組織の取引先ページを更新する。

```javascript
// データテーブルに表示する列を定義し、Columns 属性に設定する
component.set("v.Columns", [
    {label:"First Name", fieldName:"FirstName", type:"text"},  // 名（テキスト）
    {label:"Last Name", fieldName:"LastName", type:"text"},    // 姓（テキスト）
    {label:"Phone", fieldName:"Phone", type:"phone"}           // 電話（電話番号型）
]);
```

> [!ポイント] 列定義オブジェクトの3つのキー
>
> | キー | 意味 |
> | --- | --- |
> | `label` | 列の **見出し** として表示される文字列 |
> | `fieldName` | 表示するデータの **項目 API 名**（`data` 内のプロパティ名と一致させる） |
> | `type` | 値の **表示型**（`text`・`phone`・`email`・`date`・`number` など）。型に応じた書式で表示 |

> [!例] fieldName とデータの対応
>
> Apex の SOQL では `FirstName, LastName, Phone` を取得していました。列定義の `fieldName` をこれらと一致させると、各レコードの該当項目が対応する列に表示されます。一致していないとその列は空欄になります。

> [!注意] 列定義は action 呼び出しより前に置く
>
> このコードは `var action = ...` の **上** に追加します。読み込み時に列をまず用意してからサーバーからデータを取得する順序（列を定義 → データ取得）になります。

## 完成イメージ

すべて保存して取引先ページを更新すると、コンポーネントは次のように表示されます。

```text
 ┌─ Contact List for United Oil & Gas Corp ──────────────┐
 │  First Name  │  Last Name  │  Phone                    │
 │ ─────────────┼─────────────┼────────────────────────── │
 │  Edna        │  Frank      │  (212) 842-5500           │
 │  Avi         │  Green      │  (212) 842-5501           │
 │  ...         │  ...        │  ...                      │
 └────────────────────────────────────────────────────────┘
   ▲ lightning:card のタイトル（v.Account.Name）
   ▲ lightning:datatable（v.Contacts を v.Columns の列で表示）
```

> [!例] 動作の全体の流れ（おさらい）
>
> 1. 取引先ページを開く → `init` イベントで `myAction` が起動。
> 2. `myAction` が列定義（`Columns`）を設定。
> 3. `c.getContacts` をサーバーへ送信し、取引先責任者を取得。
> 4. コールバックで結果を `Contacts` 属性に保存。
> 5. `lightning:datatable` が `Contacts` を `Columns` の列定義どおりに表示。

## 試験対策：押さえておきたい追加ポイント

> [!ポイント] datatable と表示でよく問われること
>
> - `lightning:datatable` には **`data`・`columns`・`keyField`** が基本セット。
> - 列定義は `{label, fieldName, type}` のオブジェクトの **配列**。`fieldName` はデータの項目名と一致させる。
> - `type` は表示書式を決める（`phone` なら電話番号、`email` ならリンクなど）。
> - `component.set("v.Contacts", ...)` でデータを更新すると、画面は **自動的に再描画** される（リアクティブ）。
> - 表示だけの単純なケースでは、Apex を使わず Lightning Data Service で完結させることも可能。
