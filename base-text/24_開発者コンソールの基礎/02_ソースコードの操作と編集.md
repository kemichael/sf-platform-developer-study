# ソースコードの操作と編集

## 学習の目的

この単元を完了すると、次のことができるようになります。

- Apex クラスを作成する。
- Apex コードを実行する。
- Lightning コンポーネント（Aura コンポーネント）を作成する。
- Visualforce ページを作成する。

> [!ポイント] この単元のゴール
>
> 開発者コンソールで、**Apex クラスの作成と実行**・**Aura コンポーネントの作成**・**Visualforce ページの作成**という3種類の開発作業を体験します。コードの中身より、**コンソールの各メニュー（File / Debug など）の使い方**に注目しましょう。

---

## Apex コードの作成と実行

### Apex クラスの作成

Apex は組織のデータ操作やビジネスロジック追加に使う Salesforce のプログラミング言語です。たとえば取引先責任者情報が変わったときに所有者へ警告する、といった処理を書けます。

> [!用語] Apex（エイペックス）
>
> Salesforce のサーバー側で動く、**強く型付けされたオブジェクト指向**のプログラミング言語。Java に似た文法で、組織のデータ操作やビジネスロジックの追加に使います。

> [!用語] クラス（Class）
>
> 関連する処理（メソッド）やデータをひとまとめにした、プログラムの設計図のような単位。Apex では1クラスを1ファイル（拡張子 `.apxc`）として保存します。

詳細は「Apex の基礎とデータベース」モジュールを参照。ここではコードの中身より開発者コンソールの操作理解が目的です。

ここでは、飛行軌道を変更するたびに地上の宇宙飛行技術者へメールを送る Apex コードを書いてみます。

> [!手順] Apex クラスを作成する
>
> 1. **[File] | [New] | [Apex Class]** を選択する。
> 2. クラスに `EmailMissionSpecialist` という名前を付ける。
> 3. テキストエディターが開いたら、次のコードを **[EmailMissionSpecialist.apxc]** タブに貼り付け、元の内容を置き換える。

```apex
public class EmailMissionSpecialist {
   // 宛先・件名・本文を受け取ってメールを送信する
   public void sendMail(String address, String subject, String body) {
      Messaging.SingleEmailMessage mail = new Messaging.SingleEmailMessage();
      String[] toAddresses = new String[] {address};   // 宛先（配列）を組み立てる
      mail.setToAddresses(toAddresses);                // 宛先を設定
      mail.setSubject(subject);                        // 件名を設定
      mail.setPlainTextBody(body);                     // 本文（プレーンテキスト）を設定
      Messaging.SendEmailResult[] results = Messaging.sendEmail(
                                new Messaging.SingleEmailMessage[] { mail });
      inspectResults(results);                         // 結果を検査
   }
   // 送信結果を検査して成否をログ出力する
   private static Boolean inspectResults(Messaging.SendEmailResult[] results) {
      Boolean sendResult = true;
      // sendEmail は結果オブジェクトの配列を返す（このクラスは1通だけ送る）
      for (Messaging.SendEmailResult res : results) {
         if (res.isSuccess()) {
            System.debug('Email sent successfully');
         }
         else {
            sendResult = false;
            System.debug('The following errors occurred: ' + res.getErrors());
         }
      }
      return sendResult;
   }
}
```

> [!手順] 作成したクラスを保存して開き直す
>
> 1. **[File] | [Save]** で保存する。保存時にコードのエラーがチェックされる。
> 2. 構文が正しくないと **[Problems]** タブにエラーが表示される。詳細を参考に修正する。
> 3. クラスを閉じるには、タブの `EmailMissionSpecialist.apxc` 横の **[X]** をクリックする。
> 4. **[File] | [Open]** を選択する。
> 5. **[Entity Type]** 列で **[Classes]** をクリックする。
> 6. **[Entity]** 列で、開くクラスをダブルクリックする。

> [!注意] 保存時に構文チェックが走る
>
> Apex は保存時にコンパイル（構文チェック）されます。エラーがあると保存できず **[Problems]** タブに表示されるので、該当行を修正しましょう。

### Apex クラスの実行

クラスを作成したら、開発者コンソールの**実行匿名（Execute Anonymous）**機能で実行できます。コードと結果のテストに便利です。

> [!用語] 実行匿名（Execute Anonymous）
>
> クラスに保存せず、その場で書いた Apex コードを即座に実行できる機能。「匿名」とは名前付きのクラスやメソッドとして保存せずに実行する、という意味です。

> [!注意] 実行匿名は本番データに影響する
>
> 実行したコードは**すべて組織に影響します**。ここでデータを削除すると実際に削除されます。実運用組織では特に注意してください。

> [!手順] 実行匿名で Apex を実行する
>
> 1. **[Debug] | [Open Execute Anonymous Window]** を選択する。
> 2. 次のコードをコピーして貼り付ける。
> 3. `Enter your email address` を自身のメールアドレスに置き換える。
> 4. **[Execute]** をクリックし、メールが届いたか確認する。

```apex
EmailMissionSpecialist em = new EmailMissionSpecialist();   // インスタンスを生成
em.sendMail('Enter your email address', 'Flight Path Change',
   'Mission Control 123: Your flight path has been changed to avoid collision '
   + 'with asteroid 2014 QO441.');                          // メール送信メソッドを呼び出す
```

## Lightning コンポーネント（Aura コンポーネント）

### Lightning コンポーネントとは?

Lightning コンポーネントは、モバイル・デスクトップ両対応の反応型 UI を比較的簡単に構築できるフレームワークです。

> [!用語] Lightning コンポーネント / Aura コンポーネント
>
> Salesforce の画面（UI）部品を作るフレームワーク。Spring '19 以降、**Aura コンポーネント**（従来型）と **LWC** の2モデルがあります。本単元で作るのは **Aura コンポーネント**で、開発者コンソールで作成できます。**LWC は作れません**（LWC の詳細は「Lightning Web Components Basics」モジュール参照）。

開発者コンソールでは**コンポーネントバンドル**を作成できます。

> [!用語] コンポーネントバンドル（Component Bundle）
>
> コンポーネント本体とその関連リソース（スタイル・コントローラー・ヘルパー・デザインなど）をまとめた**フォルダーのような単位**。1つの部品を構成するファイル一式のセットです。

### Aura コンポーネントの作成

付近の宇宙船に挨拶して友好を示すコンポーネントを作ってみます。

> [!手順] Aura コンポーネントを作成する
>
> 1. **[File] | [New] | [Lightning Component]** を選択する。
> 2. コンポーネントに `meetGreet` という名前を付けて **[Submit]** をクリックする。アプリ／ページ等の設定オプションは無視してよい。
> 3. `meetGreet.cmp` タブをクリックする。Aura コンポーネントの開始・終了タグが表示される。
> 4. 次の行を `<aura:component>` と `</aura:component>` の間に貼り付ける。
> 5. **[File] | [Save]** で保存する。

```html
<p>Greetings, fellow humans! What’s your status?</p>
```

ウィンドウ右側のリストには、バンドルの全リソースが並びます。クリックすると対応するリソースが開き、スタイルで見た目を、ヘルパーでビジネスロジックを追加できます。

> [!例] コンポーネントバンドルのリソース構成
>
> ```text
> meetGreet（バンドル）
>  ├─ meetGreet.cmp      … コンポーネント本体（マークアップ）
>  ├─ Controller         … クライアント側のロジック
>  ├─ Helper             … 共通ロジック（ビジネスロジック）
>  ├─ Style              … 見た目（CSS）
>  ├─ Documentation      … ドキュメント
>  └─ Design / SVG …     … その他のリソース
> ```

> [!手順] 保存済みの Aura コンポーネント／リソースを開く
>
> 1. **[File] | [Open Lightning Resources]** を選択する。
> 2. 検索ボックスにコンポーネント名を入力するか、リストからフォルダーを選ぶ。
> 3. フォルダー横の矢印をクリックしてバンドルのリソースを表示する。
> 4. 作業するリソースを選択して **[Open Selected]** をクリックする。

## Visualforce ページとコンポーネントの作成

> [!用語] Visualforce（ビジュアルフォース）
>
> Salesforce 上で動く Web アプリの画面を作る**ページ中心**の開発フレームワーク。HTML に似たタグ（`<apex:...>`）で UI を記述します。

| 比較項目 | Visualforce | Lightning（Aura）コンポーネント |
| --- | --- | --- |
| 設計の中心 | **ページ中心** | コンポーネント中心 |
| 主な処理場所 | **サーバー側**（保存時に再読み込み） | 大半が**クライアント側**（デバイス上） |
| 画面更新 | サーバーとやり取りして UI を再読み込み | クライアント側で部分更新 |

> [!ポイント] Visualforce と Lightning の違い
>
> 「**Visualforce = ページ中心・サーバー側で再読み込み**」「**Lightning = コンポーネント中心・クライアント側で処理**」という対比を押さえましょう。

### Visualforce ページの作成

制御エンジニアがエンジンや燃料タンクの安全性を点検・報告するための Visualforce ページを作ります（ここでは UI の作成に焦点）。

> [!手順] Visualforce ページを作成する
>
> 1. **[File] | [New] | [Visualforce Page]** を選択する。
> 2. ページに `FlightSystemsChecklist` という名前を付ける。
> 3. 次のコードをテキストエディターに貼り付ける。
> 4. **[File] | [Save]** を選択する。
> 5. 左上の **[Preview]** をクリックすると、ブラウザーでプレビューが表示される。

```html
<apex:page sidebar="false">
<!--Flight Systems Checklist Visualforce Page-->
   <h1>Checklist</h1>
   <apex:form id="engineReadinessChecklist">
      <apex:pageBlock title="Flight Systems Checklist">
         <!--First Section-->
         <apex:pageBlockSection title="Engines">
            <!--Adding Checkboxes-->
            <apex:inputCheckbox immediate="true"/>Engine 1
            <apex:inputCheckbox immediate="true"/>Engine 2
            <apex:inputCheckbox immediate="true"/>Engine 3
            <apex:inputCheckbox immediate="true"/>Engine 4
            <apex:inputCheckbox immediate="true"/>Engine 5
            <apex:inputCheckbox immediate="true"/>Engine 6
         </apex:pageBlockSection>
         <!--Second Section-->
         <apex:pageBlockSection title="Fuel Tanks">
            <apex:inputCheckbox immediate="true"/>Tank 1
            <apex:inputCheckbox immediate="true"/>Tank 2
            <apex:inputCheckbox immediate="true"/>Tank 3
            <apex:inputCheckbox immediate="true"/>Tank 4
            <apex:inputCheckbox immediate="true"/>Tank 5
            <apex:inputCheckbox immediate="true"/>Tank 6
         </apex:pageBlockSection>
         <apex:pageBlockButtons>
            <!--Adding Save Button-->
            <apex:commandButton value="Save" action="{!save}"/>
         </apex:pageBlockButtons>
      </apex:pageBlock>
   </apex:form>
</apex:page>
```

プレビューには `form`・`pageBlock`・`inputCheckbox` などの Visualforce マークアップが示されます。

> [!手順] 保存済みの Visualforce ページを開く
>
> 1. **[File] | [Open]** を選択する。
> 2. **[Entity Type]** で **[Pages]** をクリックする。
> 3. **[Entity]** で、開くページをダブルクリックする。

## 試験対策：押さえておきたい追加ポイント

> [!ポイント] この単元の頻出ポイント整理
>
> - Apex クラスの作成は **[File] | [New] | [Apex Class]**、実行確認は **[Debug] | [Open Execute Anonymous Window]**。
> - **実行匿名（Execute Anonymous）** で実行したコードは**本番データに影響する**（削除すれば本当に消える）。
> - 開発者コンソールで作れる UI は **Aura コンポーネント**と **Visualforce ページ**。**LWC は作れない**。
> - **Visualforce = ページ中心・サーバー側**、**Lightning = クライアント側**で処理。
> - Aura コンポーネントの関連ファイル一式は **コンポーネントバンドル**としてまとまっている。
