# スキーマビルダーを使う

## 学習の目的

この単元を完了すると、次のことができるようになります。

- スキーマビルダーで既存のデータモデルを視覚化する。
- スキーマビルダーを使ってオブジェクトを作成する。
- スキーマビルダーを使って項目を作成する。

> [!ポイント] この単元のゴール
>
> **スキーマビルダー**は、オブジェクト・項目・リレーションを**1枚の図として視覚化・編集**できるツールです。複雑なデータモデルを俯瞰したり、ドラッグ＆ドロップでオブジェクトや項目を作ったりできることを押さえましょう。

---

## スキーマビルダーとは

オブジェクト・項目・リレーションが増えるとデータモデルは複雑になります。**スキーマビルダー**は、そうしたデータモデルの視覚化と編集ができるツールで、設計や理解に役立ちます。

> [!用語] スキーマビルダー（Schema Builder）
>
> オブジェクト・項目・リレーションを**図（ダイアグラム）として表示・編集**できる Salesforce のツール。オブジェクト同士のつながりを線で確認でき、ドラッグ＆ドロップで新しいオブジェクトや項目を追加できます。

> [!用語] スキーマ（Schema）
>
> データベースやアプリケーションの「構造の設計図」。どんなオブジェクトがあり、どんな項目を持ち、どう関連しているかの全体像を指します。

---

## 実際のデータモデルを表示する

> [!手順] スキーマビルダーでデータモデルを表示する
>
> 1. **[Setup（設定）]** の **[Quick Find（クイック検索）]** で `Schema Builder`（スキーマビルダー）を検索してクリックする。
> 2. 左パネルで **[選択解除]** をクリックする。
> 3. **[Contact（取引先責任者）]**、**[Favorite（お気に入り）]**、**[Offer（提案）]**、**[Property（物件）]** のチェックボックスをオンにする。
> 4. **[自動レイアウト]** をクリックする。

選択したオブジェクトがボックスで表示され、リレーションが線でつながったダイアグラムが描かれます。

```mermaid
erDiagram
    Contact ||--o{ Favorite__c : "参照関係（ゆるい）"
    Property__c ||--o{ Favorite__c : "主従関係（強い）"
    Property__c ||--o{ Offer__c : "主従関係（強い）"
    Contact {
      string Name "取引先責任者"
    }
    Property__c {
      string Name "物件（主・親）"
    }
    Favorite__c {
      string Name "お気に入り（子）"
    }
    Offer__c {
      string Name "提案（子）"
    }
```

オブジェクトはキャンバス上で**ドラッグ**できます。配置を変えてもオブジェクトやリレーションは変わりません。同僚にカスタマイズを紹介したり、データの流れを説明したりするのに便利です。

> [!例] スキーマビルダーが役立つ場面
>
> - 新メンバーに「うちの組織のデータ構造」を一目で説明したいとき。
> - 主従関係・参照関係の位置を線で確認し、削除の影響範囲を把握したいとき。
> - 設計レビューでオブジェクト間のつながりの抜け漏れを俯瞰したいとき。

> [!ポイント] レイアウト変更は安全
>
> オブジェクトをドラッグして配置を変えても、**実際のデータモデルは変わりません**（見た目の整理だけ）。一方、後述のオブジェクト作成・項目作成は**実際にメタデータを変更する**操作なので区別して覚えましょう。

```mermaid
flowchart TD
    S(["スキーマビルダーでの操作"]) --> Q{"何をする？"}
    Q -->|"オブジェクトをドラッグして配置変更"| A["見た目の整理だけ<br/>メタデータは変わらない（安全）"]
    Q -->|"要素タブからオブジェクトを追加"| B["メタデータを変更<br/>（実際にオブジェクトが作られる）"]
    Q -->|"要素タブから項目を追加"| C["メタデータを変更<br/>（実際に項目が作られる）"]
    classDef hl fill:#0176D3,stroke:#032D60,color:#fff;
    classDef soft fill:#E8F2FC,stroke:#0176D3,color:#032D60;
    class B,C hl;
    class A soft;
```

## スキーマビルダーでオブジェクトを作成する

スキーマビルダーの視覚的インターフェースからオブジェクトを作成することもできます。

> [!手順] スキーマビルダーでオブジェクトを作成する
>
> 1. 左サイドバーで **[要素]** タブをクリックする。
> 2. **[オブジェクト]** をクリックしてキャンバスにドラッグする。
> 3. オブジェクトに関する情報を入力する。
> 4. **[Save（保存）]** をクリックする。

新しいオブジェクトがスキーマビルダーに表示されます。

---

## スキーマビルダーで項目を作成する

項目の作成方法もオブジェクトと同じです。

> [!手順] スキーマビルダーで項目を作成する
>
> 1. **[要素]** タブから項目種別を選択し、作成したオブジェクトにドラッグする（**リレーション項目・数式項目・通常の項目**を作成できる）。
> 2. 新しい項目の詳細を入力する。
> 3. **[Save（保存）]** をクリックする。

オブジェクトマネージャーに戻ると、新しいオブジェクトが既存のオブジェクトと同じように表示されます。

> [!ポイント] スキーマビルダーで作れるもの・確認
>
> - スキーマビルダーから作成したオブジェクト・項目は、**オブジェクトマネージャーで作ったものと完全に同じ**です（作成方法が違うだけ）。
> - 作成できる項目種別は**リレーション項目・数式項目・通常の項目**。

## 🎓 この単元のまとめ

この単元は、データモデル全体を **1枚の図** として俯瞰・編集できる **スキーマビルダー** を扱いました。「ドラッグでの配置変更は見た目だけ（安全）」と「要素タブからの追加はメタデータを実際に変更する」の区別が核心です。

| 操作 | メタデータへの影響 | 用途 |
| --- | --- | --- |
| オブジェクトをドラッグして配置変更 | 変わらない（安全） | 図の整理・説明用 |
| 要素タブからオブジェクト追加 | **実際に作成される** | 視覚的にオブジェクト作成 |
| 要素タブから項目追加 | **実際に作成される** | リレーション・数式・通常項目を作成 |

```mermaid
flowchart TD
    SB(["スキーマビルダーを開く"]) --> V["既存データモデルを<br/>図として視覚化"]
    V --> Q{"何をする？"}
    Q -->|"配置をドラッグ"| SAFE["見た目の整理だけ<br/>メタデータ不変（安全）"]
    Q -->|"要素タブで追加"| EDIT["オブジェクト・項目を作成<br/>メタデータを実変更"]
    EDIT --> OM["オブジェクトマネージャーで<br/>作ったものと完全に同じ結果"]
    classDef hl fill:#0176D3,stroke:#032D60,color:#fff;
    classDef soft fill:#E8F2FC,stroke:#0176D3,color:#032D60;
    class SB hl;
    class V,SAFE,EDIT,OM soft;
```

> [!まとめ] この単元の要点
>
> - **スキーマビルダー**はオブジェクト・項目・リレーションを**図で視覚化・編集**できるツール。
> - オブジェクトの**ドラッグ配置変更は見た目だけ**でメタデータは変わらない（安全）。
> - **要素タブ**からオブジェクト・項目を追加すると、**実際にメタデータが変更**される。
> - 作成できる項目種別は**リレーション項目・数式項目・通常の項目**。
> - スキーマビルダーで作ったものは、**オブジェクトマネージャーで作ったものと完全に同じ**（作成方法が違うだけ）。

> [!豆知識] スキーマビルダーは「見るだけ」でも超優秀
>
> スキーマビルダーは設定変更の権限がなくても、組織のオブジェクト同士のつながり（主従／参照の線）を一望できるため、引き継ぎや設計レビューで重宝されます。リレーションの線にカーソルを合わせると関係の種類が確認でき、巨大な組織のデータモデルを把握する「地図」として使う管理者も多いです。ER 図を別ツールで描き起こす手間が省けるのが魅力です。
