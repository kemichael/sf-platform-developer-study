# まとめパート追加 + カテゴリー総まとめページ作成 仕様書（全エージェント共通）

すでに整形・図解済みの教材（`base-text/` 配下の Markdown）に対して、次の2つの作業を行う。

1. **各ユニット末尾に「まとめパート」を追加**（既存ファイルをその場で追記・上書き）。
2. **担当トピック（カテゴリー）に総まとめページを新規作成**（`99_トピック総まとめ.md`）。

全文日本語。既存の本文・見出し・コールアウト・表・コード・Mermaid 図・Challenge 設定値・API 名は **一切削らない／変えない**。**追加のみ**。

---

## お手本 / 参照
作業前に必ず次を読み、密度・記法・Mermaid の書き方を踏襲する:
- `base-text/03_数式と入力規則/01_数式項目の使用.md`（コールアウト・表・Mermaid の手本）
- `build/MERMAID_SPEC.md`（Mermaid を壊さない書き方＝**必読**）

---

## コールアウト記法（厳守）
マーカー行の **次に必ず空の `>` 行**を入れてから本文を書く。使える種別:
`[!用語]` `[!例]` `[!ポイント]` `[!注意]` `[!手順]` `[!まとめ]` `[!豆知識]`

- `[!豆知識]` … 今回の新種別。「へぇ」と思える小ネタ・由来・数値・歴史・実務Tips を1つ。**各まとめパートに最低1つ**入れる。

```
> [!豆知識] 見出し（任意）
>
> 興味を引く小ネタを2〜4行で。
```

---

## Mermaid を壊さないルール（MERMAID_SPEC より要約・厳守）
1. ノードIDは半角英数字のみ（`A` `B` `N1`）。日本語をIDにしない。
2. ラベルは必ずダブルクオートで囲む：`A["取引先 Account"]`。
3. ラベル内の改行は `<br/>`。ラベル内でダブルクオートを使わない（「」を使う）。
4. 判断分岐は `X{"条件？"}`、エッジラベルは `A -->|"はい"| B`。
5. ノード数は 5〜12 個程度。1ブロック1図。先頭で図種別を宣言（`flowchart TD` 等）。
6. 配色（任意）:
```
    classDef hl fill:#0176D3,stroke:#032D60,color:#fff;
    classDef soft fill:#E8F2FC,stroke:#0176D3,color:#032D60;
```

## 作業1：各ユニット末尾の「まとめパート」

対象トピック内の **すべての .md ファイル**（新規作成する `99_トピック総まとめ.md` を除く）。
**ファイルの最後尾**（リソース・ハンズオン Challenge も含めた全内容の後ろ）に、次の構成のセクションを追記する。

```
---

## 🎓 この単元のまとめ

<1〜2文で、この単元が何だったかを言い切る導入文>

<次のうち「この単元の核心が最も伝わるもの」を1〜2種類選んで配置>
- **図**：単元の流れ・関係・分岐を表す小さな Mermaid 図（5〜10ノード）。本文の図と重複しすぎない、俯瞰／総括の図にする。
- **表**：要点・比較・使い分けを 3〜6 行に凝縮した Markdown 表。

> [!まとめ] この単元の要点（3〜5項目の箇条書き）
>
> - …
> - …

> [!豆知識] <小ネタの見出し>
>
> この単元に関係する小ネタを1つ。
```

注意:
- 既に `[!まとめ]` コールアウトが本文中にある単元でも、**末尾のまとめパートは別途追加**してよい（重複表現は避け、図・表・豆知識で価値を足す）。
- 「簡単なまとめ」なので冗長にしない。図1つ＋要点＋豆知識で十分。表は内容に応じて。
- Mermaid が単元の性質上不要なら表だけ／要点だけでもよいが、**豆知識は必須**。

## 作業2：カテゴリー総まとめページ（新規作成）

担当トピックのディレクトリ直下に `99_トピック総まとめ.md` を新規作成する。
（ビルドはファイル名先頭の数字で並ぶため `99_` で必ずそのトピックの最後のページになる。）

タイトル（1行目 H1）は `# 〔トピック名〕 総まとめ` とする（トピック名はディレクトリ名から数字と区切りを除いたもの）。

次の要素を **視覚的に分かりやすく** 盛り込む（フル盛り構成）:

1. **導入文**：このトピックで学んだことを2〜4文で総括。
2. **全体像 Mermaid 図**：トピック全体の構造・流れ・登場概念の関係を1枚で俯瞰できる図（8〜12ノード）。`flowchart` か `mindmap` 風の `flowchart` を推奨。
3. **ユニット横断の早見表**：各ユニットの「学んだこと／キーワード／一言要点」をまとめた Markdown 表（行＝各ユニット）。
4. **`## 🎯 試験頻出ポイント`**：`[!ポイント]` で、このトピックから狙われやすい論点・暗記値を凝縮。
5. **`## 📖 用語早見表`**：このトピックで登場した重要用語を「用語｜ひとことの意味」の表で一覧化（8〜15語程度）。
6. **`[!豆知識]` を2〜3個**：トピックに関する小ネタ。
7. **`## ✅ 理解度セルフチェック`**：`[!まとめ]` 内に、Yes/No や穴埋めで答えられる確認問題を4〜6問（答えも併記）。

総まとめページは「このトピックを一気に思い出せる1枚」を目指す。図・表を主役にして読みやすく。
