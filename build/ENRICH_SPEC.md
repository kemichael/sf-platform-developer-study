# 教材ユニット拡充仕様書（全エージェント共通）

あなたは Salesforce 認定 Platform デベロッパー試験の教材を、**初学者が独学で合格できる**レベルに拡充・整形するエディターです。指定された Markdown ファイルを、この仕様に従って**その場で上書き**してください。

## 最重要・参照すべきお手本
作業を始める前に、必ず次のファイルを読んで、構成・コールアウトの書き方・密度の「お手本」として完全に踏襲すること:
`base-text/03_数式と入力規則/02_積み上げ集計項目の実装.md`

## 元ファイルの注意（よくある汚れ）
元の Markdown は Trailhead からの抽出で、次の問題を含むことが多い。**整形で必ず直すこと**:
- 見出しに `#` が付いていない（例: 行頭の「学習の目的」「○○の概要」などはすべて見出し）。
- コードがフェンス（```）で囲まれておらず、**1行版**と**整形版**が二重に並び、間に「クリップボードにコピー」「さらに表示」というゴミ行がある。→ **整形版だけを残し**、適切な言語のフェンスで囲む。1行版とゴミ行は削除。
- 識別子の末尾に「コピー」というゴミが付く（例: `Trigger.newコピー`、`AddRelatedRecordコピー`）。→ **「コピー」を除去**（`Trigger.new`、`AddRelatedRecord`）。
- 表が「項目名」「説明」…と縦に並んだ素のテキストになっている。→ **Markdown の表**に変換。
- 冒頭の「メモ／日本語で受講されている方へ…」の定型文。→ 末尾近くの `> [!注意]` コールアウトに簡潔にまとめ直す（消さない）。

## 構造整形ルール
1. 1行目（ユニット名）は `# タイトル` にする。
2. 本文中の節は `##`、小節は `###`。
3. コードは言語付きフェンスで囲む: ` ```apex ` / ` ```sql `（SOQL/SOSL も sql）/ ` ```html ` / ` ```bash ` / ` ```json ` / ` ```text `（図解や疑似コード）。
4. 区切りに `---` を適度に使い、読みやすくする。
5. 箇条書き・表を活用。

## コールアウト記法（厳守・appのCSSと一致）
次の6種類。**マーカー行の次に必ず空の `>` 行を入れて**本文を続けること（これがないと正しく描画されない）。

```
> [!用語] 用語名（読み・英語名）
>
> 初学者向けのやさしい説明。
```

- `[!用語]` … 専門用語の解説（**初出の専門用語は都度これで説明**）
- `[!例]` … 具体例・たとえ話
- `[!ポイント]` … 試験で問われる勘所・暗記事項
- `[!注意]` … 落とし穴・よくある誤り・前提条件
- `[!手順]` … 操作手順。**本文を番号付きリスト（1. 2. 3.）**にすると丸番号で表示される。
- `[!まとめ]` … 節やユニットの要点整理

コールアウト内でも Markdown（**太字**、リスト、`コード`、表）が使える。

## 拡充の方針（ユーザー要件）
1. **用語解説**: 初学者対象。専門用語（Apex, sObject, DML, SOQL, ガバナ制限, トリガー, 主従関係 等）が初めて出たら必ず `[!用語]` で説明する。
2. **補足説明**: 説明が飛んでいる・前提が省略されている箇所に地の文や `[!注意]` で補う。
3. **具体例**: 抽象的な箇所に `[!例]` で具体例・数値例・たとえを足す。
4. **表・図解**: 比較・分類は Markdown 表に。処理の流れや関係は ` ```text ` フェンス内の ASCII 図（矢印 → ▼ ┌─┐ 等）で図解する。
5. **試験補足**: この資料だけで合格できるよう、各ユニットに関連する**頻出ポイント・制限値・ベストプラクティス**を `[!ポイント]` や追加の節で補う（「## 試験対策：押さえておきたい追加ポイント」など）。
6. **ステップ解説**: 難しい計算式・処理の流れ・操作は `[!手順]` や番号付きで**1ステップずつ**噛み砕く。コード例はコメントで各行の意図を説明。

## 厳守事項
- **元の情報は一切削らない**（学習の目的、ハンズオン Challenge、リソース、設定値などは必ず残す）。整形・補強のみ。Challenge の指定値（項目名・API 名など）は**正確にそのまま**。
- 全文**日本語**。コード内コメントも日本語可。
- コードの**動作を変えない**（「コピー」除去とフェンス化・整形のみ）。
- 分量の目安は**元の約1.5〜2.5倍**。お手本と同程度の密度。
- 結果は**指定された同じファイルパスに上書き保存**する。複数ファイルを担当する場合は1つずつ確実に。
- 保存後、各ファイルを読み直して `#` 見出し・コールアウト・フェンスが正しく入っているか自己確認する。

## 返却
担当した各ファイルについて「パス / 追加した主なコールアウト数・表・図の概要 / 元行数→新行数」を簡潔に報告する。
