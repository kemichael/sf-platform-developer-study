アルファベット

カテゴリー

0 件の用語

A

Abstract Class（抽象クラス）
Apex

abstract キーワードで定義するクラス。直接インスタンス化できず、extends するサブクラスが abstract メソッドを実装する必要がある。

例： public abstract class AnimalHandler { abstract void handle(List<Animal__c> records); }

@AuraEnabled
ApexLWC

ApexメソッドをLightning Web ComponentおよびAuraコンポーネントから呼び出し可能にするアノテーション。cacheable=true を設定するとクライアントサイドキャッシュが有効になる。

例： @AuraEnabled(cacheable=true) public static List<Account> getAccounts() {...}

Apex
Apex

Salesforceのプロプライエタリなオブジェクト指向プログラミング言語。Javaに似た構文を持ち、サーバーサイドで実行される。強型付けで、ガバナー制限の下で動作する。

Anonymous Apex（匿名 Apex）
Apex

クラスやトリガーとして保存せず、Developer Console の「Execute Anonymous」やVS Code拡張、CLIから即座に実行できるApexコード。動作確認・データ修正などに使用する。本番でも実行可能（ただし注意が必要）。

Aura Component（オーラコンポーネント）
LWC

LWC の前世代のSalesforce UIコンポーネントフレームワーク。.cmpファイルと独自の JavaScript コントローラーで構成される。現在はLWCへの移行が推奨されているが、既存資産は引き続き動作する。

B

Batch Apex（バッチ Apex）
非同期ガバナー制限

Database.Batchable インターフェースを実装し、大量レコードをチャンク単位で分割処理する非同期Apex。1チャンクあたり最大200件（デフォルト）。start() → execute() → finish() の3メソッドを実装する。

制限： 一度にキューに入れられるジョブは最大5件

before / after（トリガータイミング）
トリガー

before：DML操作がDBに反映される前に実行。Trigger.new の値を変更可能。after：DBへの書き込み後に実行。レコードのIdが確定しているため参照に使用。

Bulkification（一括処理化）
データガバナー制限

SOQLクエリやDML操作をループ内に置かず、List や Map でまとめて処理するApexのベストプラクティス。一括操作によってガバナー制限への抵触を防ぐ。

NG： for ループ内で insert record; → OK： まとめて insert recordList;

C

Callout（コールアウト）
非同期

ApexからHTTP/SOAPを使って外部サービスを呼び出す機能。トリガー内で直接コールアウトは不可（@future(callout=true) や Queueable 経由が必要）。事前にリモートサイト設定で許可URLを登録する。

Custom Label（カスタムラベル）
デプロイ

組織全体で使用できる翻訳可能なテキスト値。Apexでは System.Label.LabelName、Formulaでは $Label.LabelName で参照。環境をまたいでデプロイ可能。

Custom Metadata Type（カスタムメタデータ型）
デプロイ

アプリケーション設定値をメタデータとして保存する仕組み。Custom Settingsと違い、サンドボックスから本番環境へのデプロイやパッケージングが容易。Apexから CMDT__mdt としてSOQLで取得可能。

Custom Permission（カスタム権限）
セキュリティ

独自の権限単位を定義できる機能。プロファイルやパーミッションセットに付与し、Apexでは FeatureManagement.checkPermission('PermName')、Formulaでは $Permission.PermName で参照。

Change Set（チェンジセット）
デプロイ

Salesforce組織間でメタデータをデプロイするUIベースの方法。「送信変更セット」で内容を定義し、「受信変更セット」でデプロイを実行する。事前に接続組織（Deployment Connection）の設定が必要。

注意： バージョン管理と連携できないため、大規模開発には Salesforce CLI が推奨

CPU Time Limit（CPU時間制限）
ガバナー制限

1トランザクションあたりのApexコード実行CPU時間の上限。同期処理では 10秒、非同期処理（@future / Batch / Queueable 等）では 60秒。複雑なロジックやネストされたループで超過しやすい。

CRUD（オブジェクトレベルセキュリティ）
セキュリティ

Create / Read / Update / Delete の頭文字。オブジェクトに対する基本操作権限の総称。Apexで実行ユーザーのCRUDを確認するには Schema.sObjectType.Account.isCreateable() などのメソッドを使用する。

D

Database クラス
データ

DML操作を行うシステムクラス。Database.insert(list, false) のように第2引数 allOrNone=false を指定すると部分コミットが可能（一部失敗してもロールバックしない）。Database.SaveResult でエラー内容を確認できる。

DML（Data Manipulation Language）
データ

sObjectに対するデータ操作。insert / update / delete / upsert / merge / undelete の6種類。1トランザクションあたり最大150回まで実行可能。

Dynamic SOQL（動的SOQL）
データ

クエリ文字列を実行時に組み立て、Database.query(String) で実行するSOQLの記述方法。柔軟性が高い一方、SOQLインジェクション対策として String.escapeSingleQuotes() によるサニタイズが必要。

Developer Console（開発者コンソール）
Apex

Salesforceが提供するウェブベースの統合開発環境。Apexクラス・トリガー・テストの作成・実行、匿名Apexの実行、デバッグログの確認などができる。現在はVS Code + Salesforce Extension Pack が推奨。

E

Email Service（メールサービス）
非同期

特定のSalesforceメールアドレス宛にメールを送信することで、Messaging.InboundEmailHandler インターフェースを実装したApexクラスを実行できる仕組み。メール受信をトリガーとしたデータ処理が可能。

Exception Handling（例外処理）
Apex

Apexの try / catch / finally 構文による例外処理。主な組み込み例外：DmlException（DML失敗）/ QueryException（クエリ結果が0件や複数件）/ NullPointerException（null参照）/ LimitException（ガバナー超過）。カスタム例外クラスは Exception を extend して作成する。

F

@future（フューチャーメソッド）
非同期Apex

メソッドを非同期で実行するアノテーション。引数はプリミティブ型（またはそのコレクション）のみ。戻り値は void のみ。コールアウトには @future(callout=true) を指定する。

制限： 1トランザクションあたり最大50回呼び出し可能

Flow（フロー）
自動化

Salesforceのローコードプロセス自動化ツール。Screen Flow（UI有）、AutoLaunched Flow（Apex/Processから起動）、Scheduled Flow（定期実行）などの種類がある。Workflow Rule・Process Builderの後継として移行が推奨されている。

FLS（Field Level Security / フィールドレベルセキュリティ）
セキュリティ

フィールド単位のアクセス制御。プロファイルやパーミッションセットで各フィールドの参照・編集権限を設定する。with sharing を宣言しても FLS は自動適用されない。Apexで明示的に確認するには Schema.sObjectType.Account.fields.Name.isAccessible() を使用する。

G

Governor Limits（ガバナー制限）
ガバナー制限

Salesforceがマルチテナント環境の安定性を保つためのリソース使用上限値。主要な制限：SOQLクエリ数100回、DML操作150回、ヒープサイズ6MB（同期）、CPU時間10秒（同期）、コールアウト100回。

getDescribe() / Schema.describe
データApex

オブジェクトやフィールドのメタ情報を実行時に取得する。Schema.SObjectType.Account.getDescribe() でオブジェクト情報、Schema.SObjectType.Account.fields.Name.getDescribe() でフィールド情報を取得できる。

H

Heap Size Limit（ヒープサイズ制限）
ガバナー制限

1トランザクションあたりのメモリ使用上限。同期処理では 6MB、非同期処理（@future, Batchなど）では 12MB。大量のデータをメモリ上に保持する場合に注意が必要。

I

@isTest
Apex

テストクラス・テストメソッドに付与するアノテーション。本番のガバナー制限に影響せず専用リミットを使用。Apexコードは 75%以上 のカバレッジが本番デプロイに必要。テストメソッドには @isTest または testMethod キーワードを付与する。

Interface（インターフェース）
Apex

interface キーワードで定義する型。実装クラスは implements キーワードを使い、複数のインターフェースを同時に実装可能。メソッドシグネチャのみを定義し、実装は持たない。

isInsert / isUpdate / isDelete / isBefore / isAfter
トリガー

トリガー内で使用できる Trigger クラスの静的コンテキスト変数。例：if (Trigger.isBefore && Trigger.isInsert) のように組み合わせてトリガーの実行条件を判定する。

@InvocableMethod
Apex自動化

FlowやProcess Builderから呼び出し可能にするApexメソッドへのアノテーション。@InvocableMethod(label='Get Accounts' description='...') のように label が必須。引数・戻り値は List<T> 型で定義する必要がある（コレクション対応のため）。

inherited sharing（継承共有）
セキュリティApex

呼び出し元クラスの共有モードを継承するApexのキーワード。public inherited sharing class MyClass {...} のように宣言する。キーワード省略時と同等だが、意図を明示するために宣言することが推奨される。グローバルクラスやトップレベルからの呼び出しでは without sharing として動作することがある。

L

Limits クラス
ガバナー制限Apex

現在のトランザクションでのリミット消費量を取得するシステムクラス。Limits.getQueries()（消費済みSOQL数）/ Limits.getLimitQueries()（上限値）のようにペアで使用することが多い。

LWC — Lightning Web Component
LWC

Web標準（Custom Elements, Shadow DOM, ES Modules）に基づくSalesforceのUIコンポーネントフレームワーク。HTML / JavaScript / CSS の3ファイルで構成される。Aura Componentより軽量で高パフォーマンス。

List（リスト）
Apex

順序付き可変長コレクション型。重複要素を許容する。new List<Account>() で生成。インデックスアクセス（list[0]）が可能。SOQLの結果は List<sObject> で受け取るのが基本。

Lightning Data Service（LDS）
LWC

LWCコンポーネントがApexサーバーコールなしにSalesforceレコードを読み書きできるキャッシュ付きデータアクセス層。@wire(getRecord, { recordId, fields }) などの wire adapter を使って実装する。共有キャッシュにより複数コンポーネント間でデータを効率的に共有する。

M

Map（マップ）
Apex

キーと値のペアを保持するコレクション型。Map<Id, Account> 形式でSOQLの結果をIdで高速検索する用途が多い。new Map<Id, Account>([SELECT Id FROM Account]) のショートカット初期化が便利。

Mixed DML Error
データセキュリティ

セットアップオブジェクト（User, Group, GroupMemberなど）と非セットアップオブジェクトを同一トランザクションで操作した際に発生するエラー。解決策は @future でいずれかの操作を別トランザクションに分離する。

HttpCalloutMock / MockHttpCallout
非同期Apex

テスト内でHTTPコールアウトをモック化するためのインターフェース。HttpCalloutMock を実装したクラスを作成し、Test.setMock(HttpCalloutMock.class, new MockClass()) で登録して使用する。

N

Named Credential（名前付き資格情報）
非同期

外部サービスのエンドポイントURLと認証情報（ユーザー名・パスワード・OAuth等）をSalesforceに登録する仕組み。Apexコールアウト時に callout:MyNamedCred/endpoint と書くことで、認証情報をコードにハードコードせずに済む。

Namespace（名前空間）
デプロイ

Salesforceパッケージに設定する一意識別子のプレフィックス。名前空間が設定された組織では、すべてのカスタムコンポーネントに ns__ObjectName__c のようにプレフィックスが付く。マネージドパッケージではパッケージ内要素を保護するために使用する。

O

Order of Execution（レコード保存の実行順序）
トリガー

レコード保存時の処理順序（主要部分）：
① 旧値の読み込み → ② システム検証 → ③ Apex before トリガー → ④ カスタム入力規則 → ⑤ DB保存（未コミット）→ ⑥ Apex after トリガー → ⑦ 割り当てルール → ⑧ 自動応答ルール → ⑨ ワークフロー → ⑩ コミット

OWD（Organization-Wide Default / 組織全体の共有設定）
セキュリティ

データアクセスのベースライン設定。Public Read/Write（全員が参照・編集可）/ Public Read Only（全員が参照可）/ Private（所有者のみ）の3種。OWDより広い権限は共有ルール・ロール階層で付与する。

P

Permission Set（パーミッションセット）
セキュリティ

プロファイルに追加の権限を付与するための設定セット。1ユーザーに複数のパーミッションセットを割り当て可能。プロファイルを変更せずに特定ユーザーだけに追加権限を付与する場合に使用。

Platform Event（プラットフォームイベント）
非同期自動化

__e サフィックスを持つカスタムオブジェクト。EventBusを介したパブリッシュ/サブスクライブ型のリアルタイム通信を実現する。Apexの EventBus.publish() でパブリッシュ、Apex トリガーや LWC でサブスクライブできる。

Profile（プロファイル）
セキュリティ

ユーザーに割り当てる基本権限設定。オブジェクト権限（CRUD）、フィールドレベルセキュリティ（FLS）、タブ表示設定、アプリケーション設定などを管理する。1ユーザーに1つだけ割り当て可能。

Permission Set Group（パーミッションセットグループ）
セキュリティ

複数のパーミッションセットをひとつにまとめて管理するコンテナ。ユーザーにはパーミッションセットグループを割り当てるだけで、グループ内の全権限が一括付与される。メンテナンスコストを削減できる。

Platform Cache（プラットフォームキャッシュ）
制限Apex

Apexで高コストな計算結果や頻繁に参照するデータをキャッシュする仕組み。Org Cache（組織全体で共有）と Session Cache（ユーザーセッション単位）の2種類がある。Cache.Org.put() / Cache.Org.get() で操作する。

Q

Queueable Apex
非同期Apex

System.Queueable インターフェースを実装してキューに追加する非同期Apexジョブ。@futureより柔軟で、引数にオブジェクト型を使用可能。System.enqueueJob() で起動し、ジョブのチェーン実行もサポート。

R

Role Hierarchy（ロール階層）
セキュリティ

ロール階層に基づくデータアクセス管理。上位ロールのユーザーは下位ロールのユーザーが所有するレコードに自動的にアクセスできる。OWDが Private の場合でも、ロール階層を通じてアクセスが拡大される。

Remote Site Settings（リモートサイト設定）
非同期

Apexから外部URLへのHTTPコールアウトを許可するホワイトリスト設定。[設定] → [セキュリティ] → [リモートサイトの設定] で登録する。登録していないURLへのコールアウトは CalloutException が発生する。

S

Sandbox（サンドボックス）
デプロイ

本番組織から複製したテスト・開発用の隔離環境。種類：Developer（200MB）/ Developer Pro（1GB）/ Partial Copy（5GB、一部データ）/ Full（本番と同等）。

Scheduled Apex（スケジュール Apex）
非同期Apex

System.Schedulable インターフェースを実装した定期実行Apexジョブ。System.schedule('Job Name', cronExp, new MyScheduledClass()) で登録。クーロン式で実行スケジュールを指定する。

Set（セット）
Apex

重複要素を持たないコレクション型。重複排除や contains() による存在チェック（O(1)）に使用。new Set<Id>() や new Set<String>() が典型的な使い方。

sObject
データ

Salesforceのデータモデルオブジェクトの基底型。Account、Contact、Opportunity などすべての標準オブジェクトとカスタムオブジェクト（__c サフィックス）が sObject を継承する。

SOQL（Salesforce Object Query Language）
データ

Salesforceのリレーショナルクエリ言語。基本構文：SELECT Id, Name FROM Account WHERE Industry = 'Technology' LIMIT 200。リレーションシップクエリ（親子・子親）が可能。1トランザクションあたり最大100クエリまで。

SOSL（Salesforce Object Search Language）
データ

複数のSalesforceオブジェクトをまたぐ全文検索クエリ言語。FIND 'keyword' IN ALL FIELDS RETURNING Account(Id, Name), Contact(Id, Name) のように複数オブジェクトの検索結果を一度に取得できる。

Sharing Rules（共有ルール）
セキュリティ

OWDで制限されたレコードアクセスを特定のグループ・ロールに拡張するルール。基準ベース（Criteria-based）と所有者ベース（Owner-based）の2種類がある。ロール階層ではカバーできない横断的な共有に使用する。

Static Resource（静的リソース）
デプロイ

JavaScript・CSS・画像・ZIPファイルなどをSalesforce組織にアップロードして管理する仕組み。Visualforce/LWCから参照可能。$Resource.ResourceName（Visualforce）や import myResource from '@salesforce/resourceUrl/ResourceName'（LWC）で参照する。

System.assert() / System.assertEquals()
Apex

Apexテストで期待値を検証するアサーションメソッド。System.assert(condition)：条件がtrueであることを確認。System.assertEquals(expected, actual)：2値が一致することを確認。System.assertNotEquals()：2値が異なることを確認。失敗すると AssertException が発生してテストが失敗する。

T

Test.startTest() / Test.stopTest()
Apex

Test.startTest() でガバナー制限のカウンターをリセットし、テスト本体のコードを実際の制限内で動作させる。Test.stopTest() を呼ぶと @future / Queueable などの非同期処理が強制的に実行される。

Trigger（トリガー）
トリガー

DML操作（insert / update / delete / undelete）に反応してApexコードを実行するイベントハンドラ。trigger TriggerName on ObjectName (before insert, after update) { ... } の構文で定義。1オブジェクトに1つのトリガーを推奨（TriggerHandlerパターン）。

Trigger.new / Trigger.old
トリガー

トリガー内で利用できるコンテキスト変数。Trigger.new：新しいレコードのリスト（insert/updateで使用可）。Trigger.old：古いレコードのリスト（update/deleteで使用可）。Trigger.newMap / Trigger.oldMap でIdをキーにしたMapとして取得も可能。

TriggerHandler パターン
トリガーApex

トリガーのロジックをHandlerクラスに委譲するベストプラクティス設計パターン。トリガーファイルは薄く保ち（呼び出しのみ）、ビジネスロジックはHandlerクラスに集約することでテスト・保守が容易になる。

Test.setMock()
Apex

テストクラス内でHTTPコールアウトやWebサービスコールをモックに差し替えるメソッド。Test.setMock(HttpCalloutMock.class, new MyMock()) のように呼び出し、テスト対象コードがコールアウトを実行しても実際のネットワーク通信は行わずモックレスポンスを返す。

U

upsert
データ

外部IDや標準IDに基づいて、レコードが存在しなければ insert、存在すれば update を自動判別して実行するDML操作。upsert recordList ExternalId__c; のように外部IDフィールドを指定可能。

V

Validation Rule（入力規則）
自動化

レコード保存前にFormulaで条件を評価し、条件が true の場合にエラーメッセージを表示してレコードの保存を防ぐルール。before Apexトリガーの後、DBへの書き込み前に実行される。

Virtual Class（仮想クラス）
Apex

virtual キーワードで修飾されたクラス。サブクラスが extends してメソッドを override できる。abstract クラスと違い、直接インスタンス化可能でメソッドにデフォルト実装を持てる。

Visualforce
Apex

HTMLライクなマークアップとApex Controllerを組み合わせてカスタムUIを構築する従来のUIフレームワーク。<apex:page controller="MyController"> の形式で記述。LWCが推奨される現在も、Salesforceクラシックや一部機能（PDF生成など）では引き続き使用される。

W

with sharing / without sharing
セキュリティApex

Apexクラスに適用する共有モード宣言。with sharing：実行ユーザーのOWD・共有ルール・ロール階層を適用（FLSは別途チェックが必要）。without sharing：共有設定を無視して全レコードを操作可能。宣言省略時は呼び出し元のモードを継承。

Workflow Rule（ワークフロールール）
自動化

条件を満たしたレコード保存時にアクション（メール送信・項目更新など）を実行する旧来の自動化ツール。Salesforceによる廃止が発表されており、Flowへの移行が推奨されている。

@wire
LWC

LWCでSalesforceのデータサービスや Apex メソッドをリアクティブに取得するデコレーター。@wire(getRecord, { recordId: '$recordId', fields }) record; のように書くと、依存プロパティが変わるたびに自動で再フェッチされる。プロパティか関数（data/error を受け取る）に適用可能。
